## 3. 백엔드 내부 모듈 의존성 (U2 내부)

모놀리스 백엔드 내부의 도메인 모듈 간 의존성입니다. 모든 통신은 **직접 함수 호출**로 이루어집니다.

### 3.1 모듈 간 의존성 매트릭스

| 모듈 (행 → 열에 의존) | Auth | Menu | Table/Session | Order | SSE |
|---|---|---|---|---|---|
| **Auth** | — | | | | |
| **Menu** | | — | | | |
| **Table/Session** | | | — | | ✅ |
| **Order** | | | ✅ | — | ✅ |
| **SSE** | | | | | — |

### 3.2 내부 호출 관계

| 호출자 | 대상 | 호출 방식 | 용도 |
|---|---|---|---|
| OrderService | SessionService | 직접 함수 호출 | 주문 생성 시 세션 조회/자동 생성 |
| OrderService | SSEService | 직접 함수 호출 | 주문 이벤트 발행 (생성/상태변경/삭제) |
| SessionService | SSEService | 직접 함수 호출 | 세션 종료 이벤트 발행 |
| MenuService | FileService | 직접 함수 호출 | 이미지 업로드/삭제 |

### 3.3 내부 모듈 개발 순서 (권장)

```
1. Auth 모듈 (독립 — 다른 모듈 의존 없음)
   │
   ▼
2. Menu 모듈 + Table/Session 모듈 (병렬 — 서로 독립)
   │
   ▼
3. Order 모듈 + SSE 모듈 (병렬 — Order는 Session/SSE에 의존)
   │
   ▼
4. 통합 (미들웨어, CORS, 에러 핸들링, main.py)
```

## 4. 프론트엔드 → 백엔드 통신

| 프론트엔드 | 백엔드 모듈 | 프로토콜 | 인증 |
|---|---|---|---|
| U3 (Customer) | Menu 모듈 | HTTP REST | 없음 |
| U3 (Customer) | Table/Session 모듈 | HTTP REST | 없음 |
| U3 (Customer) | Order 모듈 | HTTP REST | 없음 |
| U3 (Customer) | SSE 모듈 | SSE | 없음 |
| U4 (Admin) | Auth 모듈 | HTTP REST | 없음 (로그인) |
| U4 (Admin) | Menu 모듈 | HTTP REST | JWT |
| U4 (Admin) | Table/Session 모듈 | HTTP REST | JWT |
| U4 (Admin) | Order 모듈 | HTTP REST | JWT |
| U4 (Admin) | SSE 모듈 | SSE | JWT |

> **참고**: 모놀리스이므로 프론트엔드는 단일 백엔드 서버 주소만 설정하면 됩니다. 마이크로서비스와 달리 서비스별 base URL 설정이 불필요합니다.

## 5. 공유 리소스

| 리소스 | 공유 범위 | 위치 |
|---|---|---|
| PostgreSQL DB | U2 전체 (단일 DB 세션) | 단일 DB 인스턴스 |
| SQLAlchemy 모델 | U2 전체 | `backend/app/models/` |
| Pydantic 스키마 | U2 전체 | `backend/app/schemas/` |
| JWT 미들웨어 | U2 관리자 API | `backend/app/middleware/` |
| SSEManager | U2 전체 (인메모리) | `backend/app/services/sse_service.py` |

> **참고**: 모놀리스 구조이므로 `shared/` 디렉토리가 불필요합니다. 모든 공유 코드가 `backend/app/` 내에 위치합니다.
